# 1. 概论
## 1.1 项目
### 项目的概念

- 在**规定时间**内、**有限资源**下、**特定的环境**中，为满足某种**事先确定的要求**，而完成的一项任务或努力
- 具体的说，项目是在**一定的组织机构**中，利用**有限的资源**（如人力、物力、财力等）在**规定的时间**内所要完成的任务

### 项目的特征

1. 一次性：具有明确的**起止时间**
2. 独特性：所形成的**产品、服务**或完成的**任务**都具有一定的独特之处
3. 目的性：具有两个方面的目标 → 项目**本身**的&项目**产出物**的
4. 整体性：由一些活动**有机组成**
5. 制约性：受到各种**资源**的制约，如人力、财力、物力、时间、信息和技术等。

# 2.  项目经理、组织、团队
## 2.1 项目组织
### 定义
**项目组织**：是为完成特定的**项目任务**而建立起来的**从事项目具体工作**的组织
性质：
- 临时性
- 灵活性
### 设计原则
1. 目标性
2. 整体性
3. 指挥统一
4. 人尽其才
5. 利于控制
6. 适应性
7. 重要性
### 设计流程
1. 确定合理的项目**目标**
2. 确定项目的**工作内容**
3. 确定**各小组**目标和工作内容。
4. 项目**组织结构**设计。包括组织形式、组织层次、各层次的组织单元、相互关系框架设计等。
5. 工作**岗位**与工作**职责**确定。
6. **人员配置**。以事设岗，以岗定人。
7. 明确**工作流程**和**信息流程**。
8. 制定**考核标准**。

## 2.4 项目团队
### 定义
- 由**两个或两个**以上
- 相互**依赖**
- 承诺**共同**规则、具有**共同**愿景，愿意为**共同**的项目目标而努力的
- **互补技能**成员
- 组成的群体
### 特征
1. 专一性：为完成特定的项目而设立的组织
2. 临时性：任务完成则使命终结，团队解散
3. 双重领导：成员一般来自不同的部门或单位，既接受原单位的领导，又接受项目经理的领导
4. 团队合作：成员之间高度合作，相互信任
5. 灵活性：由于成员较少，成员在项目中所扮演的角色和所起的作用往往因环境和目标的改变而变化

### 形成阶段
1. 形成
2. 震荡
3. 规范
4. 成熟
5. 解散

### 激励原则
1. 目标原则
2. 公平原则
3. 按需激励原则
### 影响项目人员绩效考核的因素
1. 核心：领导不力、约束无力
2. 两不：目标不明、规章不全、职责不清
3. 不足：激励不足
4. 缺乏：缺乏沟通

# 4. 项目管理范围
### 项目范围
**项目范围**（Project Scope）：
- 为了成功地**实现项目目标**
- 所**必须完成**的
- **全部且最少**的工作
- 包括项目**产品范围**和**工作范围**

### 项目范围计划
- 以项目的**实施动机**为基础
- 确定**项目范围**
- 并编写**项目范围说明书**
- 的管理过程
### 工作分解技术
   思路：项目→子项目→任务→工作包→项目全部工作包
   - 按照项目范围的大小**从上到下**逐步分解
   - 首先由**项目**分解得到**子项目**
   - 再由**子项目**得到**任务**
   - 再由**任务**到**工作包**
   - 继续逐层进行分解，得到**项目全部工作包**

## 5.4 项目计划进度编制方法
### 甘特图法
难以定量计算
### 关键路径法（CPM，Critical Path Method）
> 重要：最早完工期代表任务总工期，因此==**终点EF=LF**==
#### 1. 最早时间$T_{ES}$、$T_{EF}$
- 从左往右计算
- 交汇点取**更大的**（紧前工序全部完成才能继续）
#### 2. 最晚时间$T_{LS}$、$T_{LF}$
- 从右往左计算
- 交汇点取**更小的**（需要保证紧后工序不拖延总工期）
#### 3. 总时差TF
- 代表不影响**总工期**的前提下，工序可以推迟的最多时间
- TF =$T_{LS}$-$T_{ES}$ = $T_{LF}$-$T_{EF}$
- TF=0的工序 => **关键工序**
#### 4. 单时差/自由时差EF
- 代表不影响**紧后工序最早开工**的前提下，工序可以推迟的最多时间
	- EF = 紧后工序$T_{ES}$ - 该工序$T_{EF}$

# 6. 项目成本管理
## 偏差分析法（挣值法）
### 3项二级指标
#### BCWS
项目**计划**完成工作的**预算成本**BCWS（Budgeted Cost for Work Scheduled）：是根据项目预先规定的进度和预算，在某一时点时应该完成的**工作量**和应该投入的**资金**的**累计值**。 
```
BCWS=计划工作量*单位工作量的预算成本
```
#### ACWP
项目**已完成**工作的**实际成本**ACWP（Actual Cost for Work Performed）：项目在某一时期内所完成**工作量**的**实际投入成本**
#### BCWP
项目**已完成**工作的**预算成本**BCWP（Budgeted Cost for Work Performed）：项目实施过程中实际完成工作量的预算成本，又称**挣值**。
```
BCWP=已完成工作量* 单位工作量的预算成本
```

### 4项一级指标
1. 项目成本偏差（Cost Variance）：CV=BCWP-ACWP
2. 项目进度偏差（Schedule Variance）：SV=BCWP-BCWS
3. 成本执行指数（Cost Performance Index）：CPI=BCWP/ACWP（<1 则预算超支）
4. 进度执行指数（Schedule Performance Index）：SPI=BCWP/BCWS （<1则进度滞后）

 >临界指数=SPI*CPI （<1则存在问题）
 
###  预测指标
   
完成时预算BAC（Budgeted At Completion）、完工估算EAC（Estimate at Complete），计算方法：
1. EAC=ACWP+BAC-BCWP
2. EAC=ACWP+ (BAC-BCWP)/CPI
3. EAC=BAC/CPI

# 7. 项目质量管理

## 戴明循环（PDCA）

1. P——**计划**： 
	1. 分析现状，找出**质量问题**及对象
	2. 分析产生质量问题的：**原因、影响因素**
	3. 从各种原因和因素中：确定**主要**原因、影响因素
	4. 针对质量问题的主要因素：制定改善质量的**措施及实施计划**，并预计效果
2. D—**执行**：根据计划阶段制定的计划措施，组织贯彻执行
3. C—**检查**：检查实际执行情况，并将**实施效果**与**预期目标**对比，找出存在的**问题**
4. A—**处理**：对**检查结果**进行总结和处理
